# 던전 크롤러 게임 기획서

> 턴제 / 사각형 그리드 / 3맵 / MVP + SO

---

## 1. 게임 컨셉

### 핵심 요약
- **장르**: 턴제 그리드 던전 크롤러
- **플레이**: 플레이어 턴 → 적 턴 반복, 맵 클리어 시 다음 맵 이동
- **목표**: 3개 맵 클리어, 최종 보스 격파

### 플레이어 경험
- 짧은 플레이 타임 (15~30분)
- 단순한 조작, 명확한 규칙
- 맵 진행에 따른 난이도 상승

---

## 2. 맵 구조

| 맵 | 이름 | 크기 | 적 | 특징 |
|----|------|------|-----|------|
| **맵 1** | 입구 동굴 | 8×8 | 일반 몹 1종 | 튜토리얼, 출구 찾기 |
| **맵 2** | 심층 통로 | 10×10 | 일반 몹 2종 | 함정, 아이템 |
| **맵 3** | 보스 방 | 6×8 | 보스 1종 | 보스 전용 패턴 |

### 맵 전환
- 맵1: 시작점 → 출구 타일 도달 시 맵2 진입
- 맵2: 시작점 → 출구 타일 도달 시 맵3(보스) 진입
- 맵3: 보스 격파 시 게임 클리어

---

## 3. 핵심 시스템

### 3-1. 턴 시스템
```
[플레이어 턴]
  → 이동 (인접 1칸) / 공격 (인접 적) / 아이템 사용 / 대기

[적 턴]
  → 모든 적이 순서대로 행동 (이동 or 공격)

[턴 종료]
  → 다음 플레이어 턴
```

### 3-2. 전투
- **공격**: 인접(상하좌우) 적에게 데미지
- **데미지**: 플레이어 공격력 - (선택) 적 방어력
- **사망**: HP 0 이하 시 제거 (적: 드롭, 플레이어: 게임오버)

### 3-3. 이동
- 상하좌우 4방향, 1턴에 1칸
- 벽/다른 유닛이 있으면 이동 불가

### 3-4. 적 AI (간단)
- 플레이어와 인접 → 공격
- 그 외 → 플레이어 방향으로 1칸 이동 (A* 또는 단순 추적)

## 4. 콘텐츠 목록

### 4-1. 플레이어
| 항목 | 수치 예시 |
|------|----------|
| HP | 100 |
| 공격력 | 15 |
| 이동 | 턴당 1칸 |

### 4-2. 적 (SO로 정의)
| 이름 | HP | 공격력 | 맵 | 비고 |
|------|-----|--------|-----|------|
| 슬라임 | 20 | 5 | 1, 2 | 기본 |
| 스켈레톤 | 35 | 10 | 2 | 중급 |
| 던전 보스 | 80 | 15 | 3 | 2페이즈 등 |

### 4-3. 아이템 (SO로 정의)
| 이름 | 효과 | 획득 |
|------|------|------|
| 회복 포션 | HP +30 | 맵2 바닥, 적 드롭 |
| 공격 물약 | 공격력 +5 (영구) | 보스 방 입구 |

### 4-4. 타일 종류
| 타일 | 동작 |
|------|------|
| 바닥 | 이동 가능 |
| 벽 | 이동 불가 |
| 출구 | 도달 시 다음 맵 |
| 함정 (맵2) | 밟으면 데미지 (선택) |

## 6. SO(Scriptable Object) 구조

```
ScriptableObjects/
├── MapDataSO          # 맵 크기, 타일맵 참조, 출구 위치
├── EnemySO            # HP, 공격력, 이동속도, 드롭 아이템
├── BossSO             # EnemySO 상속 or 확장, 페이즈/패턴
├── ItemSO             # 이름, 아이콘, 효과(회복량 등)
└── (선택) TileRuleSO  # 타일별 규칙
```
